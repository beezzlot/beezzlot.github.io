Добро пожаловать в Глоссарий!

Здесь вы найдете подробные объяснения ключевых терминов и понятий, связанных с Linux системами и Astra Linux. Этот глоссарий поможет вам лучше понять особенности операционных систем, их архитектуру, инструменты и принципы работы.

Ядро (Kernel)
Основной компонент операционной системы, управляющий ресурсами компьютера и обеспечивающий взаимодействие между аппаратным обеспечением и программами.
Дистрибутив (Distribution)
Комплект программного обеспечения на основе ядра Linux, включающий дополнительные утилиты, библиотеки и графическую оболочку.
Терминал (Terminal)
Текстовый интерфейс для взаимодействия с операционной системой через команды.
Оболочка (Shell)
Командный интерпретатор, который обрабатывает введенные пользователем команды и передает их ядру.
Репозиторий (Repository)
Хранилище программного обеспечения, откуда пользователь может устанавливать и обновлять программы.
Пакетный менеджер (Package Manager)
Инструмент для управления программными пакетами: установки, обновления и удаления.
Суперпользователь (Root)
Учетная запись с максимальными привилегиями, позволяющая выполнять любые действия в системе.
Демон (Daemon)
Фоновый процесс, который выполняет определенные задачи без взаимодействия с пользователем.
Файловая система (File System)
Способ организации и хранения файлов на диске.
Монтирование (Mounting)
Процесс подключения файловой системы к определенной директории в Linux.
Символьная ссылка (Symbolic Link)
Ссылка на файл или директорию, которая указывает на их местоположение.
Жесткая ссылка (Hard Link)
Ссылка на файл, которая указывает на тот же inode, что и оригинальный файл.
Inode
Структура данных, хранящая метаинформацию о файле (права доступа, владелец, размер и т.д.).
Права доступа (Permissions)
Настройки, определяющие, кто может читать, записывать или выполнять файл.
Скрипт (Script)
Файл, содержащий набор команд для автоматического выполнения задач.
Сигнал (Signal)
Сообщение, отправляемое процессу для управления его поведением (например, завершение).
Процесс (Process)
Экземпляр выполняемой программы.
Поток (Thread)
Минимальная единица выполнения процесса, которая может работать параллельно с другими потоками.
Сокет (Socket)
Интерфейс для сетевого взаимодействия между процессами.
Сеть (Network)
Набор инструментов и протоколов для взаимодействия между компьютерами.
SSH (Secure Shell)
Протокол для безопасного удаленного доступа к системе.
IPTables
Утилита для настройки сетевого экрана (firewall) в Linux.
GRUB (Grand Unified Bootloader)
Загрузчик, используемый для запуска операционной системы.
Системный журнал (Syslog)
Файл, в который записываются системные сообщения и события.
Драйвер (Driver)
Программа, обеспечивающая взаимодействие операционной системы с аппаратным обеспечением.
Вирутализация (Virtualization)
Технология, позволяющая запускать несколько виртуальных машин на одном физическом сервере.
Контейнер (Container)
Изолированная среда для запуска приложений, использующая общее ядро операционной системы.
Docker
Платформа для создания, запуска и управления контейнерами.
Bash (Bourne Again Shell)
Одна из самых популярных командных оболочек в Linux.
Скриптовая оболочка (Scripting Shell)
Использование командной оболочки для написания скриптов автоматизации.
Системный вызов (System Call)
Интерфейс для запроса услуг ядра операционной системы.
Логирование (Logging)
Процесс записи событий и сообщений в системные журналы.
Astra Linux
Российская операционная система, разработанная для защиты информации и работы с данными, имеющими гриф секретности.
Специальное ядро (Modified Kernel)
Ядро Astra Linux, адаптированное для работы в защищенных контурах и соответствия требованиям безопасности.
Мандатное управление доступом (Mandatory Access Control, MAC)
Механизм контроля доступа, используемый в Astra Linux для обеспечения высокого уровня безопасности.
Средства защиты информации (СЗИ)
Комплекс программных и аппаратных решений, встроенных в Astra Linux для защиты данных.
Гриф секретности
Уровень конфиденциальности информации, определяющий степень ее защиты.
Сертификация ФСТЭК
Подтверждение соответствия Astra Linux требованиям Федеральной службы по техническому и экспортному контролю России.
Сертификация ФСБ
Подтверждение соответствия Astra Linux требованиям Федеральной службы безопасности России.
Режим безопасности (Security Mode)
Режим работы Astra Linux, обеспечивающий защиту данных от несанкционированного доступа.
Изолированный рабочий стол (Isolated Desktop)
Функция Astra Linux, позволяющая изолировать рабочие среды для работы с данными разного уровня секретности.
Виртуализация в Astra Linux
Технология, позволяющая создавать изолированные виртуальные среды для выполнения задач с повышенными требованиями безопасности.
Аудит безопасности (Security Audit)
Процесс проверки системы на соответствие требованиям безопасности и выявление уязвимостей.
Шифрование данных (Data Encryption)
Метод защиты информации, используемый в Astra Linux для предотвращения несанкционированного доступа.
Модуль доверенной загрузки (Trusted Boot Module)
Компонент Astra Linux, обеспечивающий безопасную загрузку операционной системы.
Средства криптографической защиты (СКЗИ)
Программные и аппаратные решения, встроенные в Astra Linux для шифрования данных и обеспечения их конфиденциальности.
Доверенная среда выполнения (Trusted Execution Environment)
Изолированная среда в Astra Linux, предназначенная для выполнения критически важных задач.
Многоуровневая защита (Multi-Level Security)
Подход к защите данных, используемый в Astra Linux, который позволяет работать с информацией разного уровня секретности.
Система управления доступом (Access Control System)
Компонент Astra Linux, регулирующий доступ пользователей к данным и ресурсам системы.
Сетевые экраны (Firewalls)
Средства защиты сети, встроенные в Astra Linux для предотвращения несанкционированного доступа.
Аудит событий (Event Auditing)
Функция Astra Linux, позволяющая отслеживать и регистрировать события, связанные с безопасностью.
Контейнеры в Astra Linux
Технология изоляции приложений, используемая в Astra Linux для повышения безопасности и удобства управления.
Средства резервного копирования (Backup Tools)
Инструменты Astra Linux для создания резервных копий данных и восстановления системы.
Управление обновлениями (Update Management)
Механизм Astra Linux для установки обновлений безопасности и исправлений.
Интеграция с отечественным ПО
Возможность Astra Linux работать с российским программным обеспечением, сертифицированным для использования в защищенных контурах.
Поддержка аппаратных платформ
Совместимость Astra Linux с различными аппаратными архитектурами, включая отечественные процессоры.
Система мониторинга (Monitoring System)
Инструменты Astra Linux для отслеживания состояния системы и выявления угроз безопасности.
Средства анализа уязвимостей (Vulnerability Analysis Tools)
Программные решения в Astra Linux для поиска и устранения уязвимостей в системе.
Поддержка виртуальных машин (Virtual Machine Support)
Возможность Astra Linux работать с виртуальными машинами для создания изолированных сред.
Интеграция с СУБД
Поддержка Astra Linux работы с системами управления базами данных, включая отечественные решения.
Система управления инцидентами (Incident Management System)
Инструменты Astra Linux для оперативного реагирования на инциденты безопасности.
