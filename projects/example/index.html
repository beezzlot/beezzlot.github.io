Лабораторная работа №1

Контейниризация

Тема лабораторной работы: "Изучение и применение технологий контейниризации для развёртывания и управления приложениями"

Цель лабораторной работы: "Научиться применять технологии контейнеризации для упаковки, развёртывания и управления приложениями в различных средах, а также ознакомиться с основными инструментами и принципами работы контейнеров."

Задачи лабораторной работы:

  • Изучить основы контейнеризации и её преимущества перед традиционными методами развёртывания приложений.
  • Ознакомиться с популярными инструментами для контейнеризации, такими как Docker, Podman и другие.
  • Изучить процесс установки и конфигурации Docker на локальной машине или в виртуальной среде.
  • Научиться создавать и управлять Docker-образами и контейнерами.
  • Разработать и контейнеризировать простое приложение, используя Docker.
  • Ознакомиться с основами написания Dockerfile для автоматизации сборки контейнеров.
  • Изучить методы управления контейнерами: запуск, остановка, просмотр состояния и удаление

    • Общие теоретические сведения

    Контейнеризация — это технология, которая помогает запускать приложения изолированно от основной операционной системы. Программа упаковывается в специальную оболочку-контейнер, внутри которой — среда, необходимая для работы. Технология нужна для:

    • для изолированного запуска приложений и рабочих сред вне зависимости от системы и ПО, установленных на конкретной машине;
    • контроля ресурсов и снижения нагрузки на систему;
    • разрешения конфликтов, которые могут возникнуть из-за того, что разные приложения нуждаются в различных версиях ПО или библиотек;
    • быстрого перемещения настроенных приложений и сред с одной машины на другую;
    • создания удобной рабочей инфраструктуры;
    • ускорения процесса разработки и снижения риска ошибок;
    • масштабирования готовых решений;
    • легкого управления сложными приложениями, средами и системами.

    Update site menu

    After renaming or deleting the courses folder, you may wish to update any [[main]] menu links to it by editing your menu configuration at config/_default/menus.toml.

    For example, if you delete this folder, you can remove the following from your menu configuration:

    [[main]]
  name = "Courses"
  url = "courses/"
  weight = 50

    Or, if you are creating a software documentation site, you can rename the courses folder to docs and update the associated Courses menu configuration to:

    [[main]]
  name = "Docs"
  url = "docs/"
  weight = 50

    Update the docs menu

    If you use the docs layout, note that the name of the menu in the front matter should be in the form [menu.X] where X is the folder name. Hence, if you rename the courses/example/ folder, you should also rename the menu definitions in the front matter of files within courses/example/ from [menu.example] to [menu.<NewFolderName>].
